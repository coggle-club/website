# Coggle 数据科学社区 — 项目需求文档

## 1. 项目概述

### 1.1 项目背景

Coggle（全称 Communication For Kaggle）创立于 2019 年下半年，是国内顶尖的数据科学与数据竞赛（如 Kaggle 等）内容分享社区。本项目为 Coggle 社区的全新数字化平台，旨在提供更优质的内容展示与用户体验。

### 1.2 项目目标

构建一个集**知识展示、竞赛记录、工具导航**于一体的数据科学社区网站，覆盖桌面端与移动端的响应式体验。

### 1.3 目标用户

- 国内数据科学竞赛参与者（Kaggle、天池等平台）
- 数据科学学习者与从业者
- 对 AI/ML 技术趋势感兴趣的读者

## 3. 内容管理

### 3.1 内容存储方式

- 所有内容以 **YAML + Markdown** 文件形式存储于 `config/` 目录
- 每种内容类型（博客、教程、竞赛、模型）有独立的 YAML 元数据 + Markdown 正文
- 链接、应用、通用页面等仅 YAML 存储
- 无需数据库，内容随代码版本控制

### 3.2 内容类型

| 类型 | 元数据 | 正文 |
|------|--------|------|
| 博客文章 | `config/blog.yaml` | `config/blog/{slug}.md` |
| 教程 | `config/tutorials.yaml` | `config/tutorials/{slug}.md` |
| 竞赛 | `config/competitions.yaml` | `config/competitions/{slug}.md` |
| 模型 | `config/models.yaml` | `config/models/{slug}.md` |
| 链接 | `config/links.yaml` | — |
| 应用 | `config/apps.yaml` | — |
| 页面内容 | `config/pages.yaml` | — |

### 3.3 竞赛状态自动计算

竞赛的 `status`（ongoing/ended）不再写在 YAML 中，由后端根据 `end_date` 与当前日期自动计算：
- 有 `end_date` 且已过期 → `ended`
- 否则 → `ongoing`

## 4. 非功能需求

### 4.1 性能

- 页面加载时间 < 2s（LCP）
- 支持 ISR（Incremental Static Regeneration）和 dynamic 加载
- 图片懒加载与优化

### 4.2 SEO

- 每个页面有独立 meta 标签（title、description）
- 自动生成 sitemap.xml
- 合理的语义化 HTML 结构

### 4.3 响应式

- 桌面端：多栏布局，信息密度高
- 移动端（< 768px）：单栏布局，触屏友好
- 平板端（768px-1024px）：自适应布局

### 4.4 可维护性

- TypeScript 强类型
- 组件化开发，复用公共组件
- 清晰的项目目录结构

### 4.5 可访问性

| 要求 | 说明 | 优先级 |
|------|------|--------|
| 语义化 HTML | 使用 `<nav>`、`<main>`、`<article>` 等语义标签 | P0 |
| 图片 alt 文本 | 所有图片必须有描述性 alt 属性 | P0 |
| 键盘导航 | Tab 顺序合理、焦点可见 | P1 |
| 对比度 | 文本/背景对比度 ≥ 4.5:1 | P1 |

### 4.6 安全性

- 外部链接统一添加 `target="_blank" rel="noopener noreferrer"`
- 配置安全 HTTP Headers
- 无用户输入场景，降低 XSS 风险

## 6. 设计风格

- **配色**：以蓝色/白色为主色调（数据科学/技术感），辅以简洁的灰色层次
- **排版**：中文字体优先，正文采用无衬线字体，代码采用等宽字体
- **风格**：简洁、现代、内容优先，减少装饰性元素
- **暗色模式**：支持 ThemeToggle 切换
