# Coggle 数据科学社区 — 项目需求文档

## 1. 项目概述

### 1.1 项目背景

Coggle（全称 Communication For Kaggle）创立于 2019 年下半年，是国内顶尖的数据科学与数据竞赛（如 Kaggle 等）内容分享社区。本项目为 Coggle 社区的全新数字化平台，旨在提供更优质的内容展示与用户体验。

### 1.2 项目目标

构建一个集**知识展示、在线竞赛、工具导航**于一体的数据科学社区网站，覆盖桌面端与移动端的响应式体验。

### 1.3 目标用户

- 国内数据科学竞赛参与者（Kaggle、天池、Kesci 等平台）
- 数据科学学习者与从业者
- 对 AI/ML 技术趋势感兴趣的读者

## 3. 内容管理

### 3.1 内容存储方式

- 所有内容以 **Markdown 文件** 形式存储于项目仓库中
- 每种内容类型（博客、教程、竞赛）有独立的目录和 frontmatter 规范
- 无需数据库，内容随代码版本控制

### 3.2 内容类型

| 类型 | 目录 | 说明 |
|------|------|------|
| 博客文章 | `content/blog/` | 短篇技术分享、经验总结 |
| 教程 | `content/tutorials/` | 长篇结构化教程 |
| 竞赛 | `content/competitions/` | 竞赛介绍与复盘 |
| 页面内容 | `content/pages/` | 特殊页面（关于、工具等） |


## 4. 非功能需求

### 4.1 性能

- 页面加载时间 < 2s（LCP）
- 支持静态生成（SSG），减少服务端负载
- 图片懒加载与优化

### 4.2 SEO

- 每个页面有独立 meta 标签（title、description）
- 自动生成 sitemap.xml
- 合理的语义化 HTML 结构

### 4.3 响应式

- 桌面端：多栏布局，信息密度高
- 移动端（< 768px）：单栏布局，触屏友好
- 平板端（768px-1024px）：自适应布局

### 4.4 可维护性

- TypeScript 强类型
- 组件化开发，复用公共组件
- 清晰的项目目录结构

### 4.5 可访问性

| 要求 | 说明 | 优先级 |
|------|------|--------|
| 语义化 HTML | 使用 `<nav>`、`<main>`、`<article>` 等语义标签 | P0 |
| 图片 alt 文本 | 所有图片必须有描述性 alt 属性 | P0 |
| 键盘导航 | Tab 顺序合理、焦点可见 | P1 |
| 对比度 | 文本/背景对比度 ≥ 4.5:1 | P1 |

### 4.6 安全性

- 外部链接统一添加 `target="_blank" rel="noopener noreferrer"`
- 配置安全 HTTP Headers（X-Content-Type-Options、X-Frame-Options 等）
- 无用户输入场景，降低 XSS 风险


## 5. 技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| Next.js | 前端框架 | App Router，支持 SSG/SSR |
| TypeScript | 开发语言 | 类型安全 |
| Tailwind CSS | 样式框架 | 响应式工具类 |
| gray-matter | Markdown frontmatter 解析 | 提取元数据 |
| remark / rehype | Markdown 渲染 | MDX 内容处理 |
| date-fns | 日期格式化 | 轻量日期库 |

## 6. 设计风格

- **配色**：以蓝色/白色为主色调（数据科学/技术感），辅以简洁的灰色层次
- **排版**：中文字体优先，正文采用无衬线字体，代码采用等宽字体
- **风格**：简洁、现代、内容优先，减少装饰性元素
- **暗色模式**：支持（二期可选）
